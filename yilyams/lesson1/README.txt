Конкуретность - логическая одновременность задач/потоков. Даже если на устройстве 1 ядро,
 с помощью context-switch можно реализовать конкурентность.
Параллелизм - это реальное оновременное выполнение на разных исполнительных единицах CPU.
Аппаратная конкуретность - это количество потенциально возможных одновременных потоков. Ядра * SMT/


SMT и HT(Simultaneous Multi-Threading и Hyper-Threading) это одно и тоже. SMT - это архитектурный прием,
когда одно фактическое ядро можно имет несколько аппаратных потоков. HT имеет такое же определение, просто оно
по другому называется для intel

Процессы занимают только адресное пространство в памяти, которое хранит свое необходимое окружеие. Сами единицы CPU
занимают потоки, если все еденицы CPU заняты, то следующие потоки будут ждать пока освободится еденица CPU для выполнения.

Контекст-свитч сохраняет все рабочие регистры потока в его TCB и восстанавливает их при следующем квантe.
Поток продолжает точно c той инструкции, где был прерван; память не теряется.
Потери возможны только из-за ошибок синхронизации между потоками, а не из-за самого переключения.
